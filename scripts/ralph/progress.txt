## Codebase Patterns
- Blog index layout: Use `.blog-index-layout` grid (sidebar + main); sidebar (`.blog-sidebar`) sticky, main (`.blog-main`) for card grid; responsive: sidebar becomes horizontal strip then single column.
- Optional partial locals: In Middleman ERB partials use `defined?(variable_name)` to check for optional locals; `local_assigns` is not available (Rails-only).
- Sticky footer: Body uses `min-height: 100vh; display: flex; flex-direction: column`. Main content (e.g. `.page-content`) gets `flex: 1 1 auto`; footer gets `flex-shrink: 0` and minimal padding so it stays at bottom and content fills the rest.
- Hero-style header (non-solid): Use same background image as home (e.g. `image_path("header.jpg")`) and replicate `.hero::before`/`.hero::after` gradient overlays in a scoped class (e.g. `.page-header-hero`); keep `.wrapper` with `position: relative; z-index: 1` so text sits above overlay. Gradient bottom color must be `var(--content-bg)` (not `var(--bg)`) so it matches `.page-content` and avoids white vs off-white seam in light mode.
- Blog: Use middleman-blog; articles live in `source/blog/` as `YYYY-MM-DD-slug.html.markdown` with YAML front matter (`title`, optional `date`, `description`, `social_image`). For a single date-sorted list of local + external posts, use the `merged_blog_posts` helper (config.rb `helpers do`); render each entry with `partial "partials/blog_post_card"` and `locals: { entry: entry }`.
- Blog index: With prefix "blog", the blog index template is `source/blog/index.html.erb` (served at `/blog/`). Use `page-header` + `partial "partials/nav"` and the same `.section-label`, `.section-title`, `.cards` + card partial as the home page for consistent layout.
- Inner pages: Use `page-header` (accent background), `partial "partials/nav"` (Home + Blog), `page-title`, `page-subtitle`, then `section.page-content` with `.wrapper`. Nav partial supports multiple links; style with `.page-back` (flex gap).
- Data: Use `data/homepage.yaml` for highlights cards, `data/blog.json` for external writing links. Load in templates as `data.homepage`, `data.blog`.
- Partials: Live in `source/partials/` with leading `_`; render with `partial "partials/name"`. Card partial expects `link`, `image`, `year`, `type`, `title`, optional `description`, `image_title`, `image_attributes`, optional `tags` (array), optional `data_year`/`data_tags` for blog filter JS.
- Blog tags: Local posts use front matter `tags: [a, b]`; external posts in data.blog use `tags` (array or comma string) or `tag_list`. Helpers `entry_tags(entry)`, `all_blog_tags`, `all_blog_years` in config.rb. Blog index filter is client-side: cards have `data-year` and `data-tags` (space-separated); pills set active tag/year and JS hides non-matching cards.
- Quality: No Rubocop in Gemfile; `bundle exec middleman build` is the main verification. npm test is a placeholder (exit 1).
