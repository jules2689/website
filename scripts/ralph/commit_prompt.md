# Commit prompt

Logically chunk all current changes into separate commits:

1. **Group by intent** – Put related edits together (e.g. one feature, one refactor, one fix).
2. **One logical unit per commit** – Each commit should be a coherent, reviewable change.
3. **Write clear messages** – Use conventional style (e.g. `feat:`, `fix:`, `refactor:`) and a short, descriptive summary.
4. **Commit everything** – Every modified or new file should end up in exactly one commit; nothing left uncommitted.

Create as many commits as needed so that the history is clear and each commit stands on its own.

DO NOT push.
